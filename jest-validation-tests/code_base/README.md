# Инструкция по запуску

## Шаги для запуска проекта

1. **Перейдите в терминале в папку `code_base` и установите зависимости:**
   ```bash
   npm install
   ```

2. **После успешной установки запустите сервер разработки Vite:**
   ```bash
   npm run dev
   ```
   (скрипта "start" нет)

3. **Сервер разработки Vite запустится, и приложение откроется в браузере:**
   [http://localhost:5173](http://localhost:5173)

## Тестирование с помощью Jest

4. **Проект использует Jest для тестирования.** 
   Имеются исходные файлы с функциями валидации. Наша задача — настроить тесты для функций.

5. **При запуске через `npm test` получаем сообщение:** 
   ```
   No tests found.
   ```

6. **Тестовых файлов нет, соответственно, создаем файлы в `src/validate/`.**
   Пример: `validateCity.test.ts` и `validateDate.test.ts`.

7. **Если просто запускать тесты без настройки поддержки Jest, то будет ошибка.** 
   Есть два способа настройки:

   ### 7.1. Использование ts-jest
   Установите необходимые пакеты:
   ```bash
   npm install --save-dev jest ts-jest @types/jest typescript
   ```
   Создайте файл `jest.config.js` (модуль ESM, нельзя использовать `module.exports`) или `jest.config.cjs`.

   ### 7.2. Использование Babel
   Установите Babel:
   ```bash
   npm install --save-dev @babel/preset-typescript
   ```

8. **Jest не валидирует ваш React-код автоматически во время разработки!** 
   Вы сами пишете тесты, которые проверяют ваш код, и запускаете их.

## Важное замечание

В каждом файле теста пишите юнит-тесты на функции из соответствующего файла валидации. 

- `validateDate.ts` — исходный код функции валидации.
- `validateDate.test.ts` — файл с тестами для этой функции.

Разделяйте код и тесты — **Single Responsibility Principle**. Один файл — реализация, другой — тесты этой реализации.
